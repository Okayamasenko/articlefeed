# /discuss — 论文讨论

与用户围绕某篇已读论文深入讨论。讨论过程中主动将有价值的内容记录到笔记文件里，无需用户手动触发。

## 启动步骤

### 1. 定位笔记文件
读取 `config.json` 获取 `data_dir`（`notes_dir` = `data_dir/notes`）。

用户可以用以下任意方式指定论文：
- 文件名关键词（如 `Dong_Wang_2023`）
- 作者姓名（如 `Dong 2023`）
- 论文标题关键词

在 `notes_dir/` 下递归搜索匹配的 `.md` 文件，找到后读取笔记全文。如有同名 PDF，也一并读取以补充细节。

如果找不到匹配文件，告知用户，询问是否提供路径或先运行 `/read`。

### 2. 进入讨论
简要回顾该论文的核心内容（2-3句），然后询问用户想讨论什么。

在笔记末尾创建讨论记录小节（如该日期的小节已存在则直接使用）：

```markdown
## 讨论记录

### YYYY-MM-DD
```

### 3. 讨论过程中的主动记录规则

**每当出现以下任一情况，立刻将其追加到笔记的讨论记录小节，无需等待用户提示：**

- 用户提出了一个实质性问题，且你给出了有意义的回答
- 讨论产生了新的分析角度或洞见（对论文的，或对用户研究的）
- 澄清了某个概念的理解或适用边界
- 用户表达了"这很有用"、"这正是我需要的"等肯定性反馈，说明刚才的内容值得保留

**不记录的内容：**
- 纯澄清性的简短来回（如"你是说……吗？""对"）
- 闲聊或与论文无关的内容

**记录格式：**
```markdown
### YYYY-MM-DD

**Q：[用户问题的核心，一句话概括]**
[回答的要点，3-5句，保留关键术语和结论。如有对用户研究的具体启示，单独一段标注"→ 对论文的启示：……"]
```

多条记录依次追加，不覆盖已有内容。

### 4. 讨论结束后

当用户离开或明确结束讨论时：

**4a. 确认笔记已更新**，告知用户本次新增了几条记录。若没有产生值得记录的内容，不创建空的讨论记录小节。

**4b. 轻量更新 interest_profile.json（自动执行，无需提醒）**

读取 `ArticleFeed/interest_profile.json`，只做以下两项判断：

- **known_gaps**：本次讨论有没有揭示新的文献缺口（现有论文未覆盖的研究角度）？
  - 有 → 加入 `known_gaps`
- **known_authors**：讨论中有没有提到值得追踪的新作者（讨论延伸推荐的论文作者等）？
  - 有 → 加入 `known_authors`

**不评估 active_seed_papers**（没有读新论文，不动种子列表）。
若两项均无变化，跳过写入，不强行修改文件。
