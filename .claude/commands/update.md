# /update — 更新研究方向

研究方向、方法论、理论框架发生变化时运行。通过对话了解变化内容，同步更新所有相关配置。

---

## 使用场景

- 确定了新的研究方法（如：从定性转向混合方法）
- 聚焦到更具体的研究问题
- 加入了新的理论框架
- 研究范围扩大或缩小
- 换了关注的期刊或领域

---

## 步骤

### 1. 展示当前配置
读取 `memory/MEMORY.md`，向用户展示当前记录的研究方向摘要，确认这是他们要修改的基础。

### 2. 了解变化内容
问用户：
> 你想更新哪些方面？可以直接描述变化，不需要很正式。例如："我现在确定用访谈法了" 或 "我把研究问题缩小到只看大学生群体" 或 "我加入了批判话语分析作为框架"。

如果用户的描述比较模糊，追问一个具体问题帮助澄清。**不要一次问太多问题。**

### 3. 更新 memory/MEMORY.md
根据用户描述，精准修改对应字段：
- 只改变化的部分，不动其他内容
- 如果新增了理论框架，加入 Key Decisions 或研究摘要
- 如果 Relevant Journals 需要调整，同步更新

### 4. 重新生成 search_config.json 搜索词
根据更新后的研究方向，重新生成三个层级的搜索词：
- 保留仍然适用的旧搜索词
- 替换掉已不相关的词
- 加入反映新方向的新词
- 更新 `last_updated` 和 `update_reason`（注明本次是因研究方向更新触发）
- `based_on_notes` 保持不变

### 5. 检查 reading_list.md 分类是否需要调整
如果研究方向变化较大（如新增了方法论框架），询问用户是否需要在 reading_list.md 里新增分类。**不要自动改动已有条目。**

### 6. 汇报
- 列出本次修改了哪些内容
- 展示更新后的搜索词供用户确认
- 提示：下次运行 `/feed` 时将基于新方向推荐论文
