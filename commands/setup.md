# /setup — 初始化配置

第一次使用时运行。通过对话了解用户的研究方向，自动生成所有配置文件。

---

## 步骤

### 1. 确认路径已配置
检查 `config.json` 中 `data_dir` 是否已填写（不是占位符）。如果还没填，提示用户先完成这一步：
> 请先在 `config.json` 中填写 `data_dir` 路径，然后重新运行 `/setup`。

### 2. 收集研究信息
依次问用户以下问题（一次问一个，等待回答后再问下一个）：

**Q1：你的研究大概属于哪个方向？**
选项提示（可多选或直接描述）：
- 语言学 / 应用语言学 / 语言教育
- 社会学 / 传播学 / 媒体研究
- 心理学 / 认知科学
- 计算机科学 / AI / NLP
- 教育学 / 学习科学
- 其他

再追问：用一句话说说具体在研究什么？

**Q2：你的理论框架目前处于什么状态？**
选项提示：
- 已有明确框架（如批判话语分析、框架理论、偏见测量等）→ 请列出关键词或框架名称
- 有几个核心概念，但还没有完整框架 → 请列出关键词
- 还在探索，暂时不确定 → 跳过，之后用 `/update` 补充

**Q3：主要用什么研究方法？**
选项提示：
- 定性（访谈、话语分析、案例研究等）
- 定量（实验、问卷、统计分析等）
- 计算方法（语料库、NLP、机器学习等）
- 混合方法

**Q4：你现在处于哪个阶段？**
选项提示：
- 刚开始，还在看文献
- 有初步框架，在完善理论
- 已有数据，在分析阶段
- 在写作阶段

**Q5：有没有已经读过、觉得很重要的论文？**
列出作者和标题就好，没有的话跳过。

### 3. 生成配置文件

根据用户回答，完成以下操作：

**更新 `memory/MEMORY.md`**：
- 填写研究主题、核心变量/框架、当前状态
- 根据用户描述的领域填写 Relevant Journals

**生成初始 `search_config.json`**：
- 根据研究主题和关键词，为三个层级各生成 4-5 条搜索词
- Tier 1：核心问题关键词
- Tier 2：相同机制、不同场域的邻近研究
- Tier 3：跨领域灵感，偏技术或方法论
- 设置 `last_updated` 为今天日期，`based_on_notes` 为空数组

**生成 `data_dir/notes/reading_list.md`**：
- 根据研究方向，自动设置合适的分类（如：理论基础 / 方法论 / 实证研究 / AI相关 等）
- 如果用户在 Q5 提供了已读论文，加入"已读"分类并标记 `[x]`

**在 `data_dir/notes/` 目录下创建 `WORKFLOW.md`**：
- 从项目根目录的 `notes/WORKFLOW.md` 复制过去：`cp notes/WORKFLOW.md {data_dir}/notes/WORKFLOW.md`

### 4. 汇报
告诉用户配置已完成，展示生成的搜索词供确认，说明接下来可以：
- 输入 `/feed` 获取第一批推荐论文
- 下载 PDF 后输入 `/read` 开始精读分析
